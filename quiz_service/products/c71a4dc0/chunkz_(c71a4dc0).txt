# Why do “generations” exist?

# A useful mental model

- Traffic demand grows (voice → messaging → smartphone broadband → massive IoT)
- Spectrum is scarce → need better spectral efficiency and reuse
- Mobility + reliability requirements increase (handoff, QoS, security)
- Core networks evolve (circuit → packet → cloud-native services)

# Each generation is a bundle of:

- Radio access changes (waveform, multiple access, coding, MIMO)
- Core network changes (switching, mobility management, policy/QoS)
- New services that become “normal” (e.g., SMS, mobile web, streaming, URLLC)

Reality check: marketing labels can lag or lead the strict ITU definitions.

# Cellular idea (the “cell” abstraction)

- Split a region into many small coverage areas (cells) served by low-power base stations
- Reuse the same frequencies in non-adjacent cells to increase capacity
- Move users between cells with handoff as they travel
- Scale capacity by densifying: smaller cells, more sites, smarter interference control

# Frequency reuse (capacity foundation)

- Adjacent cells use different channels to limit co-channel interference
- A “cluster” of N cells uses a complete set of frequencies → then repeats
- Reuse factor = 1/N (smaller N = more reuse = more interference control needed)
- Key trade-off: capacity vs interference (and thus SINR and quality)

| <0 dB    | Very poor signal, high interference/noise |
| -------- | ----------------------------------------- |
| 0-10 dB  | Low, unstable connection                  |
| 10-20 dB | Moderate to good connection               |
| >20 dB   | Excellent connection, high throughput     |

# How networks scale capacity over time

- Add spectrum (new bands) and add channels where possible
- Cell splitting: smaller cells in hotspots
- Sectoring: directional antennas reduce interference (e.g., 3 sectors per site)
- Densification: microcells/femtocells; antennas on rooftops and lamp posts
- Interference coordination (ICIC/CoMP) enables tighter reuse

Interference Coordination (ICIC/CoMP) in LTE/LTE-A networks reduces inter-cell interference (ICI) to improve edge-user performance. ICIC uses frequency/power coordination via X2 interfaces, while CoMP (Coordinated Multi-Point) allows multiple base stations to coordinate transmissions, offering dynamic management for higher data rates.

# Control vs traffic: why signaling matters

- Control channels carry: synchronization, system information, paging, access requests
Paging in a mobile network is a process used to locate and establish communication with a mobile device when it is not in active use but needs to be reached for an incoming call, message, or data session.
- Traffic channels carry: user voice frames or packet data
- Capacity is limited by: both data resources and control overhead
- 5G adds: richer control for beam management and ultra-reliable access

# Power control (why it matters)

- Goal: enough power for reliable reception, but not so much that interference explodes
- Open-loop: UE estimates path loss and sets power (fast, less accurate)
- Closed-loop: network commands UE power adjustments (accurate, needs feedback)
- Critical in CDMA and uplink-limited systems (near–far problem)

# Multiple access: FDMA vs TDMA vs CDMA (intuition)

- FDMA: split spectrum into frequency channels (1G)
- TDMA: share a frequency by time slots (GSM: 8 users per 200 kHz carrier)
- CDMA: share time+frequency using spreading codes; users separated by codes
- OFDMA (later): many narrow subcarriers shared dynamically (LTE/NR)

# CDMA: why it’s different

- Frequency diversity: wideband spreading reduces frequency-selective fading impact
- Multipath resistance: RAKE combines energy from multiple paths

- Self-jamming if users aren’t well controlled
- Softer capacity limit (interference-limited)

Capacity depends on interference and required quality, not just channels.

# UMTS “cell breathing” (interference-limited)

- In CDMA systems, more users → more interference → reduced coverage
- BREATHING CELLS

# GSM

- Mobile device gets exclusive signal from the base station
- Number of devices in a cell does not influence cell size

# UMTS

- Cell size is closely correlated to the cell capacity
- Signal-to-noise ratio determines cell capacity
- Noise is generated by interference from:
- other cells
- other users of the same cell

Edge users hit power limits first and may drop, shrinking the effective cell.

- Interference increases noise level
- Devices at the edge of a cell cannot further increase their output power (max. power limit) and thus drop out of the cell
- No more communication possible

Limitation of the max. number of users within a cell required. Cell breathing complicates network planning.

Planning must account for load-dependent coverage. Cell breathing: capacity and coverage are linked in UMTS.

# 5G deployment: NSA vs SA (and DSS)
